## Phase 0 — 分发管道 ✅ 已完成

### 已交付
- **shared**：`src/cad-manifest.ts`（`CadManifest`/`CadArtifact`/`CadMcpServer` + `parseCadManifest`/`emptyCadManifest`），默认 `transport: stdio`；校验 sha256(64hex)/semver/id 唯一/mcp name 唯一/stdio 的 bundleId 必须存在于 bundles。`src/index.ts` 已 re-export。
- **gateway**：`src/codex-cad-config.ts`（`writeCodexCadMcpConfig`），幂等写 `# BEGIN/END CODEX CAD MCP` 块，与 gateway 块并存，Windows 路径用 TOML 单引号串。`package.json` 增 `./codex-cad-config` 导出。
- **admin**：`src/cad-store.ts`（manifest 持久化 + 工件落盘边算 sha256 + 引用保护）、`src/cad-routes.ts`（下列 API），已在 `buildAdminServer` 注册；CAD 数据存 `data/cad/`。

### API
- 管理端（Bearer adminToken）：`GET /api/admin/cad/manifest`、`PUT /api/admin/cad/mcp-servers`、`PUT|DELETE /api/admin/cad/artifacts/:kind/:id`（kind=skills/scripts/bundles）。
- 客户端（Bearer 客户端令牌）：`GET /api/gateway/cad/manifest`、`GET /api/gateway/cad/artifacts/:kind/:id/download`（响应头带 `x-artifact-sha256`/`x-artifact-version`）。

### 验证证据
- `npm test`：admin 17 / desktop 8 / gateway 12 / shared 11 = **48 passed**（2026-07-09）。
- `npm run build`（shared+gateway+admin，admin 含 vite web build）：通过。
- 新增测试：`packages/shared/test/cad-manifest.test.ts`(8)、`apps/gateway/test/codex-cad-config.test.ts`(6)、`apps/admin/test/cad-api.test.ts`(6)。

## 待办与交接

### windows-client（C#）接入 Phase 0（阻塞真机验收）
1. 启动代理时 GET `/api/gateway/cad/manifest`，比对本地 `revision`。
2. 逐工件校验 sha256，不符则下载并**校验响应头 `x-artifact-sha256`**后落地：skills→`~/.codex/skills/<id>/`（解压）、scripts→脚本目录、bundles→客户端私有 MCP 目录（解包）。
3. 把 `mcpServers[].bundleId` 解析为本机 `command/args`，调 `writeCodexCadMcpConfig` 写 config.toml。
4. 启动本地 MCP（stdio）。
> 复用现有下载/进度条/中文提示与令牌鉴权。
