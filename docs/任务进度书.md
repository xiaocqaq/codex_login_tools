## A — windows-client（C#）接入 Phase 0 ✅ 已完成

### 已交付（`apps/windows-client`）
- **`CadModels.cs`**：`CadManifest`/`CadArtifact`/`CadMcpServer`/`CadBundleEntry`/`CadLocalState` DTO。
- **`CadMcpConfigWriter.cs`**：幂等写 `# BEGIN/END CODEX CAD MCP` 块，照 `CodexConfigWriter` 范式，TOML 单引号串处理 Windows 路径，与 gateway 块并存。
- **`CadSync.cs`**：拉 manifest → 比对 `revision` → 逐工件**双重 sha256 校验**（内容 + `x-artifact-sha256` 响应头）→ 增量下载 → zip 解压落地（skills→`~/.codex/skills/<id>/`、scripts/bundles→本地目录）→ 解析 `bundleId`（读 bundle 内 `mcp-entry.json`）为本机 command/args → 写 config。本地状态存 `%AppData%/CodexLoginTools/cad/state.json`，支持增量与清理已删工件。
- **`MainForm.cs`**：`ToggleGatewayAsync` 启动代理后调 `SyncCadCapabilitiesAsync()`（失败仅提示、不阻断代理）；停止时 `CadSync.RemoveMcpConfig()` 移除受管块。
- **`.csproj`**：版本 `0.2.2 → 0.2.3`。

### 关键设计决策
- **stdio MCP 进程由 Codex 依 config.toml 自行拉起**，客户端只"下载+校验+落地+写配置"，不自己 spawn，简化实现且符合 Codex MCP 机制。
- **CAD 同步失败不阻断代理启动**——属可选增强能力。
- **bundle 缺 `mcp-entry.json` 时跳过该 server**，不写半截配置。

### 验证证据
- `dotnet build -c Release`：**成功，0 警告 0 错误**（net9.0-windows，2026-07-09）。

### 未验证（需真机）
- 真机端到端：拉包→config 写入→Codex 启动本地 MCP→AutoCAD 出图。依赖 Phase 2/3 产出的 bundle 与 AutoCAD 环境。

## D / Phase 3 — 本地生成/计算 MCP 🟡 核心完成

### 已交付（`apps/cad-dxf-mcp`，Python）
- **`cad_dxf_mcp/geotech.py`**：确定性计算——`rankine_ka/kp`（Rankine 土压力系数）、`layered_active_pressure`（分层主动土压力，含黏聚力受拉截断 + 地下水水土分算 + 地面超载）。纯函数，避免 LLM 直接算数。
- **`cad_dxf_mcp/drawing.py`**：ezdxf 生成——`draw_basic_geometry`（基础几何：地面/坑底/桩中心/桩底 + 标高文字 + 标准图层）、`draw_active_pressure_diagram`（土压力分布图）、`save_dxf`。
- **`cad_dxf_mcp/server.py`**：MCP 薄封装（FastMCP），暴露 4 个工具（土压力系数/分层压力表/基础几何 DXF/压力分布 DXF）。**逻辑全在已单测的 geotech/drawing**，server 只做参数转换；仅暴露确定性计算与出图，无任意代码执行。
- **`mcp-entry.json`**：bundle 启动契约（对应 A 的 CadSync 读取约定）。
- **`pyproject.toml`**：core 仅依赖 ezdxf；`[mcp]` extra 才引 mcp SDK（真机装），故本机可脱离 SDK 单测。

### 关键设计决策
- **核心逻辑与 MCP SDK 解耦**——`geotech`/`drawing` 不 import mcp，本机 pytest 即可全验；`server.py` 的工具在 `build_server()` 内惰性定义，导入不需 SDK。
- **计算做成确定性纯函数**——对应报告"计算-画图"数据孤岛痛点，杜绝抄错/算错。
- **DXF 而非 DWG**——ezdxf 不出 DWG，DWG 交本机 CAD 另存或 ODA 桥（Phase 2）。

### 验证证据
- `pytest`：**20 passed**（geotech 13 + drawing 7），基准值用 Rankine 手算校核（φ=30° → Ka=1/3, Kp=3）。
- 端到端：`layered_active_pressure` + `draw_active_pressure_diagram` + `save_dxf` 生成 DXF、可 `ezdxf.readfile` 回读。
- 环境：ezdxf 1.4.4 / pytest 9.1.1 / Python 3.13。

### 未验证（需真机）
- MCP server 层（需 `pip install "cad-dxf-mcp[mcp]"`）、PyInstaller 自包含 exe 打包、Codex 桌面端调用工具出图。
